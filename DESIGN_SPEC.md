## 一、产品概述

### 1.1 产品定位
Exponent 是一个面向"超级个体"的代币化融资平台，通过"生产力锚定"的价值发行体系，帮助保险代理人、独立开发者、内容创作者等个体完成业务扩张的资金募集。

### 1.2 核心设计理念
- **摒弃纯金融博弈**：建立基于实际业务营收的价值闭环
- **Web2/Web3 双轨融合**：降低传统用户进入门槛
- **动态流动性保护（DLP）**：智能计算融资规模与流动性配比

### 1.3 目标用户

| 角色 | 特征 | 典型场景 |
|------|------|----------|
| 超级销售 | 保险代理人、房产中介 | 需要资金购买高质量线索 |
| 独立开发者 | 程序员、AI创业者 | 需要购买算力/API |
| 内容创作者 | KOL、YouTuber | 需要资金投入内容制作 |

**融资建议公式（前端实时计算）:**

**超级销售 (Super Seller):**
```javascript
F_suggested = P_target / R_roi
// P_target: 目标利润 (用户输入)
// R_roi: 行业 ROI (默认 1.5)
```

**独立开发/AI (Solopreneur):**
```javascript
F_suggested = (U_target × C_cac) + E_fixed
// U_target: 目标用户数 (用户输入)
// C_cac: 单客获取成本 (默认 $2)
// E_fixed: 固定开支 (默认 $10,000)
```

**内容创作者 (KOL):**
```javascript
F_suggested = (W_target × C_cpf) + E_prod
// W_target: 目标粉丝数 (用户输入)
// C_cpf: 单粉获取成本 (默认 $0.2)
// E_prod: 内容制作成本 (默认 $10,000)
```

**分红快照机制:**

### 附录：项目状态机设计

#### 3.2.A.1 状态定义

| 状态码 | 状态名称 | 英文 | 进入条件 | 退出条件 |
|--------|----------|------|----------|----------|
| `preparing` | 筹备中 | Preparing | 用户创建项目 | 提交审核 |
| `pending_review` | 审核中 | Under Review | 提交审核 | 审核通过/拒绝 |
| `cornerstone` | 基石轮 | Cornerstone Round | 审核通过 | 20% 售罄或 168 小时到期 |
| `public` | 公开募集 | Public Offering | 基石轮结束 | 达硬顶或 90 天到期 |
| `coming` | 即将上市 | Coming Soon | 达硬顶 | 审核通过，资金解锁 |
| `listed` | 已上市 | Listed | 资金解锁完成 | 毕业或失败 |
| `failed` | 募集失败 | Failed | 90 天未达硬顶 | 资金退回 |

**注：** "募集失败"状态仅在后端记录，不在前端展示。

#### 3.2.A.2 状态流转图

```
                                    90 天到期未达硬顶
                                         ┌────────┐
                                         │        │
                                         ▼        │
┌──────────┐   ┌──────────┐   ┌──────────┴───┐   │
│preparing │ → │pending_  │ → │ cornerstone  │   │
│ 筹备中   │   │ review   │   │ 基石轮      │   │
└──────────┘   │ 审核中   │   └──────────┬───┘   │
               └──────────┘              │        │
                     │                   │ 168h 到期/售罄
                     │ 拒绝              │
                     ▼                   ▼
               ┌──────────┐   ┌──────────────────┐
               │ rejected │   │    public        │
               │  拒绝    │   │   公开募集       │
               └──────────┘   └──────────┬───────┘
                                         │
                              ┌──────────┼──────────┐
                              │          │          │
                              ▼          ▼          ▼
                        ┌────────┐  ┌────────┐  ┌────────┐
                        │ coming │  │ failed │  │ listed │
                        │即将上市│  │ 失败   │  │ 已上市  │
                        └────────┘  └────────┘  └────────┘
```

#### 3.2.A.3 各状态详细说明

**1. preparing (筹备中)**
- 前端展示：项目可见但不可投资
- 用户操作：可编辑项目信息、提交审核
- 触发条件：用户创建项目后

**2. pending_review (审核中)**
- 前端展示：项目不可见
- 审核周期：24-48 小时
- 用户操作：可撤回修改

**3. cornerstone (基石轮)**
- 前端展示：项目可见，仅邀请码持有者可投资
- 价格：固定低价（前 20% 份额）
- 持续时间：168 小时（7 天）或售罄
- 退出条件：20% 份额售罄或时间到期

**4. public (公开募集)**
- 前端展示：项目可见，所有人可投资
- 价格：联合曲线指数上升
- 持续时间：最长 90 天
- 退出条件：达到硬顶或时间到期

**5. coming (即将上市)**
- 前端展示：项目可见但不可投资
- 状态说明：等待资金解锁审核
- 触发条件：达到硬顶后

**6. listed (已上市)**
- 前端展示：项目可交易
- 状态说明：正常交易状态
- 退出条件：毕业（完成分红目标）或失败

**7. failed (募集失败)**
- 前端展示：项目下线
- 触发条件：90 天未达硬顶
- 处理：资金原路退回给投资者

### B1. DLP核心数学模型

这是 Exponent 的财务总线。系统不询问用户"你想发多少币"，而是询问"你需要多少钱干活"。

#### 基础 DLP 公式 (简化版)

$$
L = \max(L_{min}, \quad F \times \beta)
$$

$$
H = F + L
$$

#### 完整 DLP 公式 (含角色系数)

$$
L = \max(L_{min}, \quad \beta \cdot H, \quad \frac{H}{1 + \gamma \cdot \frac{F}{H}})
$$

$$
H = F \times (1 + \beta + \beta \times \gamma)
$$

#### 变量定义

| 变量符号 | 名称 | MVP设定/范围 | 设定理由 |
|----------|------|--------------|----------|
| $F$ | 融资需求 (Funding Need) | 用户填写 ($5k - 500k$ U) | 个体用于购买线索、算力、硬件的实际现金 |
| $H$ | 募集硬顶 (Hard Cap) | **系统自动计算** | 内盘募集的上限金额 |
| $L$ | 入池资金 (Liquidity) | **系统自动计算** | 毕业时锁入 Uniswap 的资金 |
| $L_{min}$ | 保底流动性 | **$5,000$ USD** | 维持"尊严线"，防止开盘暴跌 |
| $\beta$ | 比例系数 | **25%** | 确保市值/流动性比处于健康区间 |
| $\gamma$ | 生产力系数 | 角色相关 (0.8-1.8) | 反映不同角色的现金转化速度 |

### B2. 角色模板与融资建议公式

超级个体在入驻时选择标签，录入业务预期，系统自动给予融资建议值。

#### 超级销售 (Super Seller)

| 属性 | 值 |
|------|-----|
| **生产力系数** | $\gamma = 1.8$ |
| **适用人群** | 保险代理人、房产中介、高端销售顾问 |
| **特征** | 现金转化极快，增长幅度较少 |

**前端建议公式：**
$$
F_{suggested} = \frac{P_{target}}{R_{roi}}
$$

- $P_{target}$：超级个体设定的 1 年目标利润（如 12 万美金）
- $R_{roi}$：该行业的预估投产比（保险行业默认 **1.5**）

**UI 话术：** "你想在今年赚到 12 万美金佣金？按行业平均 1:1.5 的线索/投放回报率，建议您募集 **8 万美金** 用于购买精准客户线索。"

#### 独立开发/AI (Solopreneur)

| 属性 | 值 |
|------|-----|
| **生产力系数** | $\gamma = 1.2$ |
| **适用人群** | 独立程序员、AI创业者、SaaS开发者 |
| **特征** | 需买算力/API，有一定研发周期，投入大，增幅预期大 |

**前端建议公式：**
$$
F_{suggested} = (U_{target} \times C_{cac}) + E_{fixed}
$$

- $U_{target}$：1年目标获取的新用户数（如 200,000 人）
- $C_{cac}$：预估单客获取成本（如 2 美金/人）
- $E_{fixed}$：1 年的固定开支（服务器、AI API 预存等，默认 10,000 美金）

**UI 话术：** "想要获取 20 万名核心用户并维持服务运转？涵盖预估获客成本与服务器开支，建议您募集 **41万美金** 作为启动弹药。"

#### 内容创作者 (KOL)

| 属性 | 值 |
|------|-----|
| **生产力系数** | $\gamma = 0.8$ |
| **适用人群** | 网红/KOL、YouTuber、自媒体创作者 |
| **特征** | 现金流不稳，变现周期中等，投入小 |

**前端建议公式：**
$$
F_{suggested} = (W_{target} \times C_{cpf}) + E_{prod}
$$

- $W_{target}$：目标新增粉丝数（如 50 万粉）
- $C_{cpf}$：预估单粉获取成本（Cost Per Fan，0.2 美金/粉）
- $E_{prod}$：年度内容制作与服务外包，设备成本（如 1 万美金）

**UI 话术：** "向 50 万粉丝目标冲刺！涵盖高质量视频制作与早期投流预算，建议您募集 **11 万美金** 彻底解放您的创作生产力。"

### B3. L/B 流动性机制详解

#### L 部分 (Liquidity Amount)

| 属性 | 说明 |
|------|------|
| **物理形态** | 钱（USDT 或 ETH） |
| **来源** | 从总募集资金 $H$ 中切出 |
| **作用** | 作为底仓资金维持交易深度 |
| **计算** | $L = \max(L_{min}, F \times \beta)$ 或完整公式 |

L 是募集来的钱里，必须牺牲掉用于维持市场交易深度的"成本"。

#### B 池 (Liquidity Reserve Pool)

| 属性 | 说明 |
|------|------|
| **物理形态** | 代币（总量 15%，即 1.5 亿枚） |
| **来源** | 系统创世时强制预留 |
| **作用** | 毕业时与 L 配对建池 |
| **优势** | 保护创始人原始股不被稀释 |

B 池是系统为了建池子，提前准备好的"公款代币"。

#### 毕业上市流程

```
1. 切分 L（钱）
   从总募集 $H$ 中切出 $L 用于流动性

2. 提取 B（币）
   从 B 池提取与 $L 等值的代币

3. 配对锁死
   将 $L (钱) + 等值代币 (B) 注入 Uniswap

4. 通缩销毁
   B 池剩余代币直接打入黑洞销毁
```

**通缩机制说明:** 如果配对后 B 池有剩余（如只用了 5000 万枚，剩余 1 亿枚），剩余部分会被**直接销毁**。这对所有持币者是巨大利好，因为流通量减少了。

### 二、缺陷修复

#### 1. 滚动条消失问题

```css
html body[data-scroll-locked] {
  overflow: visible !important;
  margin-right: 0 !important;
}
```

#### 2. 语言切换不工作

**解决方案**: 使用 Next.js router 的 `replace` 方法：
```typescript
const router = useRouter();
const pathname = usePathname();
router.replace(pathname, { locale: value });
```

**相关文件**: `components/language-switcher.tsx`

#### 3. 暗黑/亮色模式不生效

**相关文件**: `app/globals.css`

#### 4. X.com Logo 错误
